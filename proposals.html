Section 01

Introduction

The Dados Abertos API from Câmara dos Deputados exposes legislative data publicly with no authentication. All requests are plain HTTP GET. Responses are JSON.

ℹ️
Base URL All endpoints in this guide are relative to https://dadosabertos.camara.leg.br/api/v2. The API does not require API keys or tokens.

A proposição (proposal) is any legislative document formally presented to the Câmara: a Projeto de Lei (PL), a Proposta de Emenda à Constituição (PEC), a Medida Provisória (MPV), and many others.

Once submitted, a proposal moves through tramitações — a sequential log of every action taken on it (committee reviews, votes, approvals, rejections, archiving). Understanding this chain tells you the full lifecycle of any bill.

Section 02

How it works

Here is the typical sequence of calls needed to go from a list of proposals all the way to knowing who authored them and what happened at each stage.

1

List proposals

Call GET /proposicoes with date filters and sorting to get a paginated list. Each item returns an id.

2

Get proposal details

Use the id to call GET /proposicoes/{id} for the full ementa, current status, and related documents.

3

Fetch authors

Call GET /proposicoes/{id}/autores to list every deputy or entity who signed the proposal.

4

Follow the tramitação

Call GET /proposicoes/{id}/tramitacoes to get an ordered log of all legislative events: who evaluated it, when, and with what outcome.

5

Check voting records

Call GET /votacoes?idProposicao={id} to retrieve official vote tallies from sessions where the proposal was voted on.

Section 03

List proposals

Returns a paginated list of proposals matching the given filters. The results support filtering by date, type, keyword, author, and more.

GET

/proposicoes

https://dadosabertos.camara.leg.br/api/v2/proposicoes

Returns summary objects for each matching proposal. Use ordenarPor=dataApresentacao&ordem=DESC to get the most recent first.

Parameter Type Required Description
dataApresentacaoInicio string optional Start date filter — format YYYY-MM-DD
dataApresentacaoFim string optional End date filter — format YYYY-MM-DD
siglaTipo string filter Type of proposal: PL, PEC, MPV, PDC, REQ, etc.
numero integer optional Official number of the proposal within its year
ano integer optional Year the proposal was submitted (e.g. 2026)
idDeputadoAutor integer filter Filter by a specific deputy's numeric ID
autor string filter Partial name search across all authors
siglaPartidoAutor string filter Author's party acronym — e.g. PT, PL, PSDB
keywords string filter Free-text keyword search in the ementa and subject tags
codSituacao integer filter Current status code — see the Status reference section below
ordenarPor string optional Field to sort by: dataApresentacao, id, ano, numero
ordem string optional ASC or DESC — default is ASC
itens integer optional Results per page — max 100, default 15
pagina integer optional Page number for pagination — default 1

Section 08

Status reference

Key values you will encounter in descricaoTramitacao and descricaoSituacao fields — and what they mean in practice.

✅ Approval keywords

Aprovada — passed by a committee or the full Plenary
Aprovada com Substitutivo — approved with amendments
Sancionada / Promulgada — signed into law

❌ Rejection keywords

Rejeitada — voted down by a committee or Plenary
Prejudicada — moot due to another decision
Vetada — vetoed by the Executive after passing

⏳ In-progress keywords

Aguardando Designação de Relator — waiting for a rapporteur
Em Pauta — on the agenda for a session
Em Votação — currently being voted on

🗄️ Archiving keywords

Arquivada — shelved at end of legislature
Retirada pelo Autor — withdrawn by the lead author
Encerrada — process fully closed

ℹ️
Full reference codes To get the complete machine-readable list of situation codes, call: GET /referencias/proposicoes/codSituacao. This returns every cod + nome pair you can use as a codSituacao filter.
Section 09

Recipe: latest proposals today

A complete JavaScript snippet showing how to fetch the 20 most recent proposals submitted in 2026. 

// JavaScript — fetch latest proposalsconst BASE = "https://dadosabertos.camara.leg.br/api/v2";

async function getLatestProposals({ tipo = "PL", itens = 20 } = {}) {
  const params = new URLSearchParams({
    siglaTipo: tipo,
    dataApresentacaoInicio: "2026-01-01",
    ordenarPor: "dataApresentacao",
    ordem: "DESC",
    itens,
  });

  const res = await fetch(`${BASE}/proposicoes?${params}`);
  const { dados } = await res.json();
  return dados;
}

// Usage
const proposals = await getLatestProposals();
console.log(proposals.map(p => `${p.siglaTipo} ${p.numero}/${p.ano} — ${p.ementa}`));
Section 10

Recipe: proposals by a specific deputy

// JavaScript — find a deputy then get their proposalsasync function getProposalsByDeputy(deputyName) {
  // 1. Find the deputy
  const depRes = await fetch(`${BASE}/deputados?nome=${encodeURIComponent(deputyName)}&itens=5`);
  const { dados: deps } = await depRes.json();

  if (!deps.length) throw new Error("Deputy not found");
  const deputy = deps[0];

  // 2. Get their proposals
  const params = new URLSearchParams({
    idDeputadoAutor: deputy.id,
    ordenarPor: "dataApresentacao",
    ordem: "DESC",
    itens: 20,
  });

  const propRes = await fetch(`${BASE}/proposicoes?${params}`);
  const { dados: proposals } = await propRes.json();

  return { deputy, proposals };
}

const { deputy, proposals } = await getProposalsByDeputy("Tabata Amaral");
console.log(`${deputy.nome} — ${proposals.length} proposals found`);
Section 11

Recipe: full proposal history

Given a proposal ID, fetch its details, all authors, and the complete tramitação — in parallel. 

// JavaScript — fetch everything about a proposal at onceasync function getFullProposal(id) {
  const [detailRes, authorsRes, tramRes] = await Promise.all([
    fetch(`${BASE}/proposicoes/${id}`),
    fetch(`${BASE}/proposicoes/${id}/autores`),
    fetch(`${BASE}/proposicoes/${id}/tramitacoes`),
  ]);

  const [{ dados: detail }, { dados: authors }, { dados: events }] =
    await Promise.all([detailRes.json(), authorsRes.json(), tramRes.json()]);

  // Last tramitação = current / final status
  const lastEvent = events.at(-1);
  const isApproved = lastEvent?.descricaoTramitacao
    ?.toLowerCase().includes("aprovad");

  return {
    proposal: detail,
    authors,
    history: events,
    currentStatus: lastEvent?.descricaoSituacao,
    approved: isApproved,
  };
}

// Usage
const full = await getFullProposal(2650431);
console.log(full.currentStatus);   // → "Aprovada"
console.log(full.authors[0].nome); // → "Alencar Santana"
console.log(full.history.length);  // → 7 (steps in tramitação)
